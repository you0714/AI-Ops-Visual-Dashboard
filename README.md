# AI-Ops-Visual-Dashboard 智能运维指挥中心大屏
基于 Vue3 + Vite + TypeScript + ECharts 开发的AI运维可视化实训大屏，面向运维业务场景，完整还原智能运维指挥中心数据可视化界面，可用于课程实训、项目答辩、前端可视化学习。

## 🖥️ 项目预览

![智能运维指挥中心大屏](./docs/screenshots/智能运维指挥中心.png)
整体为16:9深色科技风运维大屏，完整覆盖运维全维度监控模块：顶部全局概览区、核心业务指标卡片、多维度业务图表、节点负载监控、系统稳定性仪表盘、运营能力雷达、实时告警日志。

## 📖 项目简介
本项目为AI运维实训配套可视化大屏，模拟7×24小时全链路运维监控场景，整合业务流量、节点负载、系统健康度、全域节点拓扑、实时告警联动等运维核心数据模块。
项目采用纯前端Mock模拟运维业务数据，无需后端接口即可完整运行演示；项目分层架构规范，预留真实业务API对接入口，支持快速替换线上数据源，适合前端数据可视化课程作业、毕设、实训汇报。

## ✨ 核心功能与页面模块
### 1. 顶部全局概览区
- 项目标题：智能运维指挥中心，附带场景说明「实时业务态势·全局事件编排·自动化决策支持」
- 全局运维摘要：全局服务健康度、自动联动响应统计、巡检/AI诊断状态、同步周期
- 右上角实时信息：系统当前时间、数据源标识、联动节点总数、重点业务数量、平均响应时长

### 2. 四大核心指标卡片（实时业务统计）
- 今日访问量：展示当日总访问次数，对比昨日涨幅
- 实时订单数：实时在线订单总量，环比昨日数据
- 活跃用户数：平台在线活跃用户规模
- 系统健康度：全局服务稳定健康百分比，环比波动

### 3. 流量分布模块 - 业务链路占比饼图
拆分教学访问、实训任务、案例下载、直播互动四类业务流量占比，直观展示业务流量倾斜情况。

### 4. 中枢视图模块 - 全域拓扑态势图
可视化运维中枢与北京、上海、深圳、成都、武汉五大业务节点数据链路流转关系，动态展示节点联动拓扑。

### 5. 实时折线图 - 吞吐量趋势监测
全天00:00-21:00吞吐量时序曲线，标记流量峰值窗口，实时展示业务吞吐变化。

### 6. 节点压力模块 - 地域负载排行横向柱状图
北京、上海、深圳、杭州、成都、武汉各地域节点负载排行，识别高负载业务节点。

### 7. 系统稳定性模块 - 健康度环形仪表盘
可视化展示全局系统健康度核心指标，直观反馈平台整体稳定状态。

### 8. 能力评估模块 - 运营能力五维雷达图
从图表设计、前端工程、交付规范等多维度评估平台运维运营能力。

### 9. 事件快照模块 - 告警联动日志
滚动展示各区域、业务平台实时告警时间戳，还原运维实时事件联动场景。

## 🛠 技术栈
- 前端框架：Vue 3 + Vite + TypeScript
- 可视化图表：ECharts
- 全局状态管理：Pinia
- 网络请求：Axios（分层服务封装）
- 模拟数据：本地Mock静态运维数据集
- 代码规范：ESLint + Prettier
- 开发环境：VSCode

## 🚀 本地启动步骤（VSCode）
### 1. 克隆/打开项目
```bash
# 拉取GitHub仓库代码
git clone https://github.com/you0714/AI-Ops-Visual-Dashboard.git
# 进入项目根目录
cd AI-Ops-Visual-Dashboard
```
VSCode操作：顶部菜单栏「文件」→「打开文件夹」，选中 `AI-Ops-Visual-Dashboard` 文件夹加载项目。

### 2. 安装项目依赖
```powershell
# 切换国内镜像加速下载依赖
npm config set registry https://registry.npmmirror.com
# 完整安装依赖包
npm install
```
> 提示：若npm命令报错，需先安装Node.js LTS版本，安装完成后重启VSCode终端。

### 3. 启动本地开发服务
项目固定端口 `10001`，自动监听本地127.0.0.1地址
```powershell
npm run dev
```
启动成功终端输出：
```
VITE v8.1.3  ready in 459 ms
➜  Local:   http://127.0.0.1:10001/
```
浏览器访问 `http://127.0.0.1:10001` 即可查看智能运维指挥中心大屏，修改代码实时热更新。

## 🔄 数据源切换说明
项目默认加载本地Mock运维模拟数据，开箱即用无需后端；
如需对接真实业务后端接口，修改环境配置切换数据源模式：
1. 在项目根目录新建 `.env.production`
2. 填入后端接口地址：
```env
VITE_DATA_SOURCE=api
VITE_API_BASE_URL=http://你的后端服务地址
```
所有图表、指标统一调用services请求层，无需修改页面业务代码。
