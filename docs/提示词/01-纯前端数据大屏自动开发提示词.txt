这是一个公开开源的教学型数据大屏项目，主要用于帮助学生和初学者从 0 到 1 学习如何自己动手制作一个数据可视化大屏。

核心要求：
1. 当前阶段只实现纯前端。
2. 数据必须使用 mock 数据。
3. 后期必须可以平滑切换到真实 API。
4. 不能把所有代码堆在一个文件中。
5. 必须遵循模块化开发规范。
6. 必须包含测试系统。
7. 必须包含日志系统。

页面设计要求：
请实现一个 1920x1080 风格的数据大屏页面，能够自适应浏览器窗口。

页面主题：
科技感、深色背景、蓝青色视觉风格，但不要只使用单一颜色。整体要专业、清晰、适合教学演示。

页面内容建议：
- 顶部标题：如意数据大屏 RuyiBigScreen
- 当前时间
- 核心指标卡片，例如：
  - 今日访问量
  - 实时订单数
  - 活跃用户数
  - 系统健康度
- 中间区域：
  - 地图或中心态势总览图
- 左侧区域：
  - 访问趋势折线图
  - 分类占比饼图
- 右侧区域：
  - 城市排名柱状图
  - 能力雷达图
- 底部区域：
  - 实时动态列表或告警列表

数据要求：
请创建 mock 数据，不要写死在组件里。

mock 数据应该包含：
- summary metrics
- trend data
- category distribution
- ranking data
- radar data
- activity list

数据访问规范：
页面组件不能直接读取 mock 文件。
必须通过 services 层获取数据。

请实现统一的数据源切换机制：

- 当环境变量 VITE_DATA_SOURCE=mock 时，使用 mock 数据。
- 当环境变量 VITE_DATA_SOURCE=api 时，预留 Axios API 请求逻辑。
- 当前默认使用 mock。
