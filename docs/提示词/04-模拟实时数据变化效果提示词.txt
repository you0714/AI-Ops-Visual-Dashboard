该模块负责：

2. 基于上一帧数据生成下一帧数据。

4. 自动生成实时动态和告警消息。

数据变化要求：

一、顶部指标卡 summary

当前指标包括：

- 今日访问量
- 实时订单数
- 活跃用户数
- 系统健康度

实时变化建议：

1. 今日访问量：
   - 每次刷新递增。
   - 增量可以在 20 到 300 之间随机。
   - 不要出现下降。

2. 实时订单数：
   - 大多数情况下递增。
   - 可以小幅波动，但不能剧烈跳变。
   - 增量可以在 0 到 80 之间随机。

3. 活跃用户数：
   - 可以上下浮动。
   - 波动范围建议控制在当前值的 1% 到 3%。
   - 不能小于 0。

4. 系统健康度：
   - 保持在 95 到 99.9 之间。
   - 大多数时候为 good。
   - 偶尔下降到 96 左右时，可以触发 warning 动态。

5. trend 字段要根据变化合理更新，不要只改 value 不改 trend。

二、访问趋势 trend

当前 trend 数据包含：

```ts
{ time: string, visits: number, orders: number }
```

请实现滑动窗口效果：

1. 每隔几秒追加一个新的时间点。
2. 新时间点使用当前时分秒或时分格式。
3. 保留最近 8 到 10 个点。
4. 移除最旧的点。
5. visits 和 orders 要与 summary 中的访问量、订单数保持大体一致。
6. 折线图刷新时不要空白或报错。

三、实时动态与告警 activities

当前 activities 数据包含：

```ts
id
 time
 level
 message
```

请实现动态消息滚动效果：

1. 每 4 到 6 秒新增一条动态。
2. 新消息插入到列表最前面。
3. 只保留最近 5 到 8 条。
4. level 从 info、success、warning 中选择。
5. warning 不要太频繁，大约 15% 到 25% 概率即可。
6. 消息文案围绕教学数据中心场景，不要写成电商、物流或金融大屏。

推荐消息方向：

- 北京学习中心完成新一轮访问数据同步
- 可视化案例库同步 12 条练习记录
- 学员端新增一批项目实战提交
- 教师端发布新的课堂任务
- 数据质量巡检通过，异常值已自动标记
- 华东节点接口延迟轻微升高，已切换备用通道
- 告警中心发现轻微波动，正在持续观察
- 问答互动区新增高频问题聚类结果

四、如意数据中枢 hubNodes

当前 hubNodes 业务节点包括：

- 课程学习
- 项目实战
- 资料下载
- 问答互动
- 教师端
- 学员端
- 数据服务
- 告警中心

实时变化建议：

1. 每次刷新时让节点 value 小幅波动。
2. 波动范围建议为 -4 到 +5。
3. value 保持在 40 到 100 之间。
4. value >= 70 时 status 通常为 good。
5. value 在 55 到 69 之间时可以为 warning。
6. value < 55 时可以为 danger，但 danger 要很少出现。
7. coord 不要变化，避免节点位置乱跳。
8. description 可以保持不变，也可以根据状态轻微变化。

五、城市访问排名 ranking

实时变化建议：

1. 每 8 到 10 秒变化一次。
2. 每个城市访问量小幅递增。
3. 变化后重新排序。
4. 排名可以偶尔发生变化，但不要每次大洗牌。

六、分类占比 categories

实时变化建议：

1. 每 10 到 15 秒变化一次。
2. 各分类占比轻微变化。
3. 总和保持为 100。
4. 不要出现负数。
5. 不要让饼图频繁剧烈跳动。

七、能力雷达模型 radar

实时变化建议：

1. 默认低频变化，30 秒左右变化一次即可。
2. 每项能力值只做 1 到 3 分的小幅变化。
3. 不要频繁刷新，避免显得不真实。
4. 也可以选择保持不变，但需要在代码中说明原因。

刷新机制要求：

页面进入时：

1. 立即加载一次数据。
2. 启动定时刷新。
3. 建议每 2 秒刷新一次 dashboard 数据。

页面离开时：

1. 必须清理定时器。
2. 避免内存泄漏。
3. 避免重复进入页面后出现多个定时器同时刷新。

推荐在 Pinia store 中增加刷新控制，而不是在组件里堆复杂逻辑。

例如可以在 store 中提供：

```ts
startRealtime()
stopRealtime()
```

或者在 DashboardView.vue 中使用简单定时器，但实时数据生成逻辑必须放在 mocks 或 services 层，不要写在组件里。
