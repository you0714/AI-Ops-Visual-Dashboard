这是一个公开开源的教学型数据大屏项目，主要用于帮助学生和初学者从 0 到 1 学习如何自己动手制作一个完整的数据可视化大屏。

推荐实现方式：

新增独立截图脚本：

```text
scripts/capture-dashboard.mjs
```

不要把它写进 E2E 测试文件里。E2E 测试负责验收，截图脚本负责生成展示资产。

package.json 中新增脚本：

```json
"screenshot": "node scripts/capture-dashboard.mjs"
```

如果需要，也可以增加更明确的脚本名：

```json
"screenshot:dashboard": "node scripts/capture-dashboard.mjs"
```

截图脚本行为要求：

1. 自动创建目录：

```text
docs/screenshots/
```

2. 默认访问地址：

```text
http://127.0.0.1:10001/
```

3. 设置浏览器 viewport：

```text
1920x1080
```

4. 等待页面标题出现，例如：

```text
如意智能教学数据中心
```

或根据当前页面实际标题等待对应 heading。

5. 等待核心大屏内容出现：

- 指标卡片
- 如意数据中枢
- `data-hub-chart` 图表容器

6. 等待 3 到 5 秒，让 ECharts 和模拟实时数据刷新至少一轮。

7. 检查页面是否有明显加载失败状态。

8. 收集浏览器 console error。

9. 截图保存到：

```text
docs/screenshots/dashboard-1920x1080.png
```

10. 截图应使用当前视口截图，不要截成长图。

推荐 Playwright 截图参数：

```ts
await page.screenshot({
  path: 'docs/screenshots/dashboard-1920x1080.png',
  fullPage: false,
})
```

11. 截图完成后，在终端输出截图路径。

12. 如果存在 console error：

- 打印错误列表。
- 保留已经生成的截图。
- 使用非 0 exit code 退出。

13. 如果没有 console error：

- 打印成功信息。
- 使用 0 exit code 退出。
